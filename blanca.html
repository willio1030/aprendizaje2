Introduccion Git

* Git es un sistema que administra o controla las distintas versiones de un programa.a lo largo de la evolucion del programa

es una aplicacion openSourse, desarrollado poro linus torvalds para controlar las contribuciones en el kernel de linux

* detalla un listado de cambios que se hallan realizado en el programa

* es un sistema de control distribuido: permite la cordinacion con otros usuarios para la alteracion del codigo por medio de copias locales de cada participante, concervacion de cambios y posterior adicion al programa principal en el repositorio remoto o local

* permite la utilizacion de registros de versiones anterior: Git registra cada linea de codigo y ademas es utilizable en cualquier version del programa

* ayuda a la reversion de cambios en el programa y control de los usuarios que realizaron cambios

* Git trabaja con repositorios locales y remotos

* colabora a capturar modificaciones y navegar en los cambios que se realizaron

actua directamente sobre el sistema operativo asi que se utilizara la consola (git bash) que posee variadas y algunas mejoradas funcionalidades

Estadios de trabajo en Git

Una aplicacion administrado por Git pasa por tres estadios:

* permite trabajar, seguir las modificaciones

* agregar archivos para prepararlos para guardar cambios

* agregar cambios al repositorio

working directory

es el area de trabajo donde se trabaja con la version local y demas archivos necesarios para el proyecto

Staging area

se agregan todos los archivos que se pretenden subir al principal

repository

Cuando los cambios son confirmados, los archivos del staging area se procede a guardar en el repositorio
